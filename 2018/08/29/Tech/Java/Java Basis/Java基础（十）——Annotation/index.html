Java基础（十）——Annotation

Aug 29, 2018 | Java Basis

本章所说的Annotation是代码里的一种特殊标记，它是一个接口。程序可以通过反射来获得指定程序元素的Annotation对象，然后通过Annotation对象获得注解里的元数据。
访问和处理Annotation的工具统称APT（Annotation Processing Tool）。

基本Annotation

java.lang下

@Override：限定重写父类方法

它只能修饰方法，不能修饰其它程序元素

@Deprecated

表示某个程序元素（类、方法等）已过时.

@SuppressWarnings

抑制编译器警告，指示该程序元素取消显示指定的编译器警告。
如@SuppressWarning(value=”unchecked”);

堆污染警告和@SafeVarargs

堆污染（heap pollution）：将一个不带泛型的对象赋给一个带泛型的变量时发生的情况。
使用@SafeVarargs修饰引发该警告的方法或构造器，是Java7专门为“抑制”堆污染警告提供的。

函数式接口与@FunctionalInterface

如果接口中只有一个抽象方法（可以包含多个默认方法和static方法），该接口就是函数式接口。@FunctionalInterface用来指定某个接口必须是函数式接口。

JDK的元Annotation

java.lang.annotation包下的元Annotation（Meta Annotation）
其中有5个元Annotation都是为了修饰其它的Annotation：@Retention、@Target、@Documented、@Inherited

自定义Annotation类

定义Annotation

使用@interface关键字，与定义接口的语法非常像。

定义了Annotation之后，就可以在程序的任何地方使用该Annotation，使用Annotation的用法非常类似于public、final这样的修饰符，可以用于修饰类、方法、变量、接口等定义。通常将Annotation另起一行。

提取Annotation信息

使用Annotation修饰了类、方法、成员变量等成员后，这些Annotation不会自动生效，需要开发者提供相应的工具来提取并处理Annotation信息。
java.lang.reflect包下的AnnotatedElemen接口。
